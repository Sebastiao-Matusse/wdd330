Description

As we move into the next sprint of the project, some new requirements have been added. Your team should add these as cards to your Trello board and complete them over the next three lessons (1.5 weeks). Each team should plan on completing the Team card plus at least one card per person per lesson (two each week).

Changes and additions

Product search
Add a search box to the navbar that will allow you to search for a product. When the form submits, the search results should be retrieved from the API and displayed on a product list page.
Product list sort
On the product list page, add the ability to sort by name or price.
Duplicate items in cart
Currently our addToCart method does not check to see if an item is already in the cart before adding it again. Make this check...if the item is in the cart do not add it again.
Correct image size
Right now our pages show the same sized product image for all browser window sizes. Change the image shown based on the screen width.
Sale flag
Add a visual indicator of how much a product is discounted to the product listing page.
Cart quantities
Add the ability to change the quantity of a product desired in the cart.
Breadcrumbs
Right below the navbar add a breadcrumb that will indicate where the user is in the site. On the home page it will not show. On the product list it will show "Product Category->(#products)" (so if you were looking at Tents it would be "Tents->(24 items)". If you were looking at a product it would show "Product category"

The first time a visitor comes to the site it should show them a banner or modal that encourages them to register with the site, and gives details about a giveaway for those who register.
Quick lookup
On the product listing page, add a quick view button. When clicked, it should show a modal with the product details in it.
Discount flag on product detail
Add a flag on the product detail page indicating the amount the product is discounted.
Newsletter sign up.
Add a box on the home page that would allow a user to sign up to receive a Newsletter.
Team 4: Dynamic header and footer
As we move forward, the site would benefit greatly from dynamic generation of any pages dealing with products. We began this process last week. Now we should finish by addressing the product listing page, as well as allowing for the templating of the header and footer.
Team 5: Expand inventory
With the changes we have made, we are ready to expand our inventory into 4 categories: Tents, Backpacks, Sleeping Bags, and Hammocks. The products can be accessed through a couple of API endpoints. /products/search/{search term} and /product/id. The main index page should be updated to show the four categories. Clicking on one should take you to a new product listing page for the appropriate category.
Team 6: Checkout
Add a button to the cart page to take the user to a checkout form. Add the form to gather the required info (see the team activity instructions in I-Learn) calculate the shipping, tax, and order total, then submit the order to the server.

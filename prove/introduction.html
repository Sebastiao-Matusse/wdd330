Description

Very few applications are written by one developer. Almost all are built by a team, or multiple teams. There are two major branches of web development currently: Frontend development and Backend development. This course (WDD 330) As the name would indicate focuses on the frontend. This is code that runs in the browser. Essentially HTML, CSS, and Javascript.

Backend development involves code that runs on a server. There are many different languages that can be used. Here at BYUI we have courses in the following backend languages: PHP, NodeJS, .NET, Python, and Java. Two of these: CSE 340 and CSE 341 reflect well the two major approaches to the backend currently:

  • CSE 340 approaches it from a monolithic standpoint. Most of the business logic runs on the server, the HTML is generated by the server and sent to the browser. Any Javascript will be to add interactivity. This is often called Server Side Rendering (SSR)
  • CSE 341 uses a more client oriented approach. The backend is used for data persistance, authentication, and other similar things. These are exposed as API endpoints or services. The Frontend contains most of the business logic and will retrieve data from the backend, then generate the HTML necessary to display it. This is called Client Side Rendering (CSR)

This course uses the second approach (client side), and as such is a very nice match with CSE 341.

The Challenge

For the final challenge of this course your team will be matched with another team made of students currently taking CSE 341. The two teams need to work together to design and build a full backend and frontend for a web application. For this week you will have the following tasks:

  1. Get to know the other team you will be working with. The instructors will make the cross course team assignments and let you know who you will be working with. You should establish how you will stay in communication with each other. (There will be a cross course Team set up with a channel for your team if you want to use it.)
  2. Brainstorm a list of application ideas that sound interesting (5-10). These ideas should require a backend and frontend. There should be a need for a database and user management of some sort (which means authentication).

As you are brainstorming start with the interests of the team members. Building an application about something you are familiar with (and enjoy!) will be easier and more fun than something you are not interested in.

You are not required to select one idea at this point (though that will need to be done by Week 6), but if there is one that really stands not and the team is excited about go ahead and indicate that it will likely be your choice. Submit the list of ideas in this activity.
